# packages

این پوشه برای بسته‌های مشترک و قابل استفاده مجدد در کل مونورپو در نظر گرفته شده است.

## هدف
- نگه‌داری قراردادها، نوع‌ها، shared utilities و پکیج‌های قابل استفاده در چند اپ
- جلوگیری از تکرار کد و افزایش سازگاری میان ماژول‌ها

## ساختار پیشنهادی
- packages/shared: DTOها، enums، قراردادهای مشترک
- packages/ui-tokens: توکن‌های UI و طراحی سیستم
- packages/database: Prisma schema و client

## نکته معماری
این بسته‌ها باید سبک، مستقل و بدون وابستگی‌های پرت به سرویس‌های خاص باشند.
