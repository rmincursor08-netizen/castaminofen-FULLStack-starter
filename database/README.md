# database

این پوشه برای پشتیبانی داده‌های پروژه، مهاجرت‌ها، اسکریماها و ابزارهای پایگاه داده در نظر گرفته شده است.

## هدف
- نگه‌داری schema، migrations، seed و مدل‌های داده
- جداسازی منطق داده از منطق برنامه

## ساختار پیشنهادی
- database/prisma
- database/migrations
- database/seed
- database/backups

## نکته معماری
مدل داده باید به‌صورت نسخه‌دار، کنترل‌شده و قابل بازتولید مدیریت شود.
