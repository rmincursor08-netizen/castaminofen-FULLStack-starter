# Tooling

This folder holds repository-level tooling, automation, and developer experience assets.

## Purpose
- standardizes linting, formatting, build orchestration, and code generation
- supports consistent contributor workflows
- improves reliability and onboarding

## What belongs here
- build and task configuration
- linting and formatting conventions
- repo automation helpers and scripts wrappers

## Design rule
Tooling should make development easier, not more complex.
