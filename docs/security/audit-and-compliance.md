# Audit Logs and Compliance

## Audit logs
- record authentication events, password changes, role changes, permission grants, and sensitive mutations
- capture actor, target, action, timestamp, origin, and outcome
- store logs immutably and retain them according to policy

## Device management
- list trusted devices and sessions
- allow signed-out sessions, revoke access, and mark unknown devices
- support risk-based re-authentication

## Security headers
- Strict-Transport-Security
- X-Content-Type-Options
- Referrer-Policy
- Permissions-Policy
- Content-Security-Policy
- X-Frame-Options
