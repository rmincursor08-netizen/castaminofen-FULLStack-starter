# Docs

This folder contains project documentation, architecture references, contributor guidance, and product knowledge.

## Purpose
- centralizes technical and product documentation
- supports onboarding and long-term maintainability
- preserves architecture decisions, development workflows, and standards

## What belongs here
- architecture docs and ADRs
- developer and contributor guides
- product audit and implementation roadmap material

## Design rule
Documentation should be clear, versioned, and easy to navigate for both engineers and stakeholders.
