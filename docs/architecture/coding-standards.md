# Coding Standards

The codebase should remain readable, typed, and consistent across packages and apps.

## Standards
- Prefer TypeScript and explicit types.
- Keep functions focused and composable.
- Avoid hidden state and unnecessary side effects.
- Favor shared abstractions over duplicate logic.

## Review expectations
- Use clear naming and small, well-scoped changes.
- Keep comments focused on intent rather than implementation trivia.
- Ensure new code fits the surrounding module boundaries.
