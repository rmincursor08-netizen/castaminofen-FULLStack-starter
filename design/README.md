# design

این پوشه برای مستندات طراحی، wireframeها، مدل‌های UX/UI و منابع طراحی محصول در نظر گرفته شده است.

## هدف
- ثبت ایده‌ها و طراحی تجربه کاربری و رابط کاربری
- اتصال میان محصول، طراحی و پیاده‌سازی

## ساختار پیشنهادی
- design/system
- design/ux
- design/ui
- design/brand

## نکته معماری
طراحی باید یک مرجع مشترک برای توسعه‌دهنده، طراح و ذینفعان باشد تا هماهنگی بهتر ایجاد شود.
